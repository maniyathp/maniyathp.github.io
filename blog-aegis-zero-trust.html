Building AEGIS: A Zero Trust Framework for Critical Infrastructure

Jul 2026 · Pranav Maniyath

Most conversations about Critical National Infrastructure (CNI) security focus on the network perimeter — firewalls, segmentation, intrusion detection at the edge. AEGIS started from a different question: what happens once an attacker is already inside the browser session of someone with legitimate access?

The problem with perimeter-only thinking

Zero Trust says "never trust, always verify" — but most implementations stop verifying once a session is authenticated. AEGIS treats the browser itself as a trust boundary that needs continuous, not one-time, verification. That means watching for anomalous behavior after login: unusual data access patterns, exfiltration attempts, session hijacking signatures.

Why explainable AI matters here

Anomaly detection models are only useful to an operator if they can explain themselves. A black-box model that flags "suspicious activity" without justification gets ignored or disabled under alert fatigue. AEGIS uses Explainable AI (XAI) techniques so every flagged event comes with a human-readable reason — which features triggered the anomaly score, and how confident the model is.

Federated learning for distributed sites

CNI environments are rarely a single network — they're distributed across sites that can't (and shouldn't) share raw data with each other or a central server. Federated learning lets each site train locally and share only model updates, so detection improves system-wide without centralizing sensitive operational data. This also keeps the design aligned with NCIIPC and CERT-In compliance expectations around data sovereignty.

Where this goes next

The current build targets deployment scenarios for Indian national defence and critical infrastructure ecosystems. Next steps are hardening the data exfiltration prevention module and running the anomaly detector against adversarial test cases to see where the explanations break down.
